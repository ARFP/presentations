# C’est quoi un algorithme ?

Un **algorithme**, c’est :

> **Une suite d’instructions simples, ordonnées, permettant de résoudre un problème.**

Exemples du quotidien :
- suivre une recette de cuisine  
- faire un trajet avec un GPS  
- monter un meuble IKEA  
- trier son linge  

➡️ **Pas besoin de connaître l’informatique pour comprendre un algorithme.**

---

# Pourquoi apprendre les algorithmes ?

- Pour **décomposer un problème** en petites étapes
- Pour **raisonner logiquement**
- Pour **préparer l’apprentissage du code**
- Parce que tous les langages (HTML, JS, PHP…) reposent sur ces mêmes idées

---

# Les briques de base d’un algorithme

1. **Les instructions**  
   → actions simples : afficher, calculer, demander une valeur…

2. **Les variables**  
   → des boîtes où l’on stocke des informations

3. **Les conditions**  
   → “si… alors…”

4. **Les boucles**  
   → répéter une action plusieurs fois

---

# Exemple simple : faire un café

**Objectif :** préparer un café avec une machine à capsules.

**Algorithme :**

1. Prendre une tasse  
2. Mettre la tasse sous la machine  
3. Insérer une capsule  
4. Appuyer sur le bouton  
5. Attendre que le café coule  
6. Boire le café ☕

➡️ C’est un algorithme : une suite d’étapes claires et ordonnées.

---

# Exemple informatique (sans coder)

**Problème :** demander l’âge d’une personne et dire si elle est majeure.

**Algorithme :**

1. Demander l’âge  
2. Lire l’âge  
3. Si l’âge ≥ 18  
   → afficher “Vous êtes majeur”  
4. Sinon  
   → afficher “Vous êtes mineur”

➡️ Même logique que dans la vie réelle.

---

# Exemple de boucle

**Objectif :** compter de 1 à 5.

1. Définir un compteur à 1  
2. Tant que le compteur ≤ 5  
   → afficher le compteur  
   → ajouter 1 au compteur  

➡️ On répète la même action jusqu’à une condition d’arrêt.

# Exercices

## Exercice 1 : Décrire un algorithme du quotidien  
Écrire les étapes pour réaliser **un sandwich simple**.  
Objectif : 6 à 10 étapes maximum.

Questions d’aide :  
- Quelles sont les actions ?  
- Dans quel ordre ?  
- Y a-t-il des conditions ? (ex : “si je veux du fromage…”)  

---

## Exercice 2 : Mini-algorithme informatique  
Écrire un algorithme qui :

> Demande un nombre à l’utilisateur et dit si ce nombre est **positif**, **négatif** ou **égal à zéro**.

Indications :  
- 1 variable pour stocker le nombre  
- 3 cas possibles  
- Utiliser des “si… alors…”
