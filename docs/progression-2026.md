# Vers une Pédagogie "Product-First" 
## Transformer l'apprentissage du code en création de valeur

---

## 1. Le constat : L'ancienne approche 
### Un apprentissage en silos "décousus"

- **Janvier :** Algorithmique pure (théorique)
- **Février :** POO (abstrait)
- **Mars :** BDD (sans lien visuel)
- **Avril :** On essaie de tout lier... péniblement.

> **Résultat :** L'étudiant sait faire une boucle `for`, mais il ne sait pas pourquoi elle est utile pour son inventaire de chaussures.

---

## 2. La Nouvelle Vision 👟
### On ne code pas pour coder, on code pour livrer

Au lieu d'apprendre des outils, on apprend à construire des **User Stories (US)**.

**Le cycle de vie d'un module :**
1. **Besoin :** "En tant qu'admin, je veux gérer mes stocks."
2. **Conception :** Wireframe -> Logigramme -> Données.
3. **Implémentation :** Front + Back + BDD en parallèle.

---

## 3. Comparaison des flux 🔄

| Ancienne (Technocentrée) | Nouvelle (Produit) |
| :--- | :--- |
| Focus sur l'outil (PHP, SQL...) | Focus sur la fonctionnalité (Panier, Login) |
| On attend 3 mois pour voir le projet | Le projet commence au Jour 1 |
| Apprentissage linéaire | Apprentissage itératif (Agile) |
| "Comment ça marche ?" | "À quel besoin je réponds ?" |

---

## 4. La Méthodologie : Le Bloc A-B-C

Pour chaque fonctionnalité, on déroule systématiquement :

- **Bloc A (Analyse) :** Comprendre l'US et définir les données.
- **Bloc B (Conception) :** Maquette (Mobile First) et Logique.
- **Bloc C (Production) :** Code réel et intégration.

---

## 5. Pourquoi ça marche ?

1. **Sens :** L'apprenant voit l'utilité immédiate de chaque ligne de code.
2. **Autonomie :** En maîtrisant la conception (Bloc B), il devient capable de résoudre des problèmes complexes.
3. **Réalité métier :** C'est ainsi que travaillent les meilleures agences et startups (Product-Led).

---

## Conclusion 

> "Le code n'est qu'un détail d'implémentation. Le produit est la finalité."
