# 🚀 Nouvelle Approche : Produit > Code
## Structurer la pensée des DWWM par le langage

---

## 🚩 Le problème : Le "Flou Artistique"
**Ancienne approche :** L'étudiant code vite, mais nomme mal.
- Les variables s'appellent `data`, `info`, `tab`.
- L'User Story est imprécise : *"L'utilisateur voit le truc."*

**Résultat :** Un code difficile à relire et des bases de données incohérentes.

---

## 🎯 La Solution : Le "Unified Process" enrichi
On garde vos étapes, mais on verrouille le **langage** dès le départ.

### Étape 1 : User Stories & Lexique 📘
Avant de dessiner ou de coder, on définit les mots interdits et les mots officiels.

> **Règle d'or :** Si un mot n'est pas dans le lexique, il n'existe pas dans le projet.

---

## 🛠️ Workshop 01 : E-Commerce
**Le défi :** Éviter l'amalgame autour du mot "Article".

- **Action :** L'étudiant doit séparer le catalogue du stock.
- **Le Lexique officiel :**
  - **Produit :** La fiche visible par le client (titre, photo, prix).
  - **Unité :** L'objet physique en entrepôt (numéro de série, poids).
- **Impact US :** *"En tant que Client, je veux voir le **Produit**."* (et non "le truc").

---

## 🎮 Workshop 02 : Jeu Vidéo
**Le défi :** Préciser les actions (les verbes).

- **Action :** Décomposer le verbe "Attaquer".
- **Le Lexique officiel :**
  - **Dégâts :** Retrait de points de vie.
  - **Animation :** Effet visuel à l'écran.
  - **Usure :** Perte de durabilité de l'arme.
- **Impact Algo :** On ne crée pas une fonction `attaquer()`, mais trois fonctions précises.

---

## 🏗️ Notre Workflow DWWM (Bloc A & B)

1. **CADRAGE :** Besoin client.
2. **USER STORIES + LEXIQUE :** Qui ? Quoi ? (Mots officiels). ⬅️ *Nouveau*
3. **MAQUETTAGE :** UI/UX utilisant les noms du lexique.
4. **SCÉNARIOS :** Parcours utilisateur avec le lexique.
5. **SÉQUENCE :** Échanges techniques (API/Données).
6. **ALGORITHMES :** Logique en pseudo-code.

---

## ✅ Les avantages pour l'apprenant

- **Zéro syndrome de la page blanche :** Les noms des variables et des tables sont déjà dans le lexique.
- **Professionnalisme :** Ils parlent comme des experts métier, pas comme des techniciens isolés.
- **Préparation au CDA :** Le lexique est la première marche vers le DDD qu'ils verront plus tard.
